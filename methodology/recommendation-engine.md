## 1. Overview

The BSCRI recommendation engine translates assessment scores into actionable recommendations. It bridges the gap between knowing where you stand and knowing what to do about it.

### 1.1 Purpose

The recommendation engine serves three purposes:

| Purpose | Description |
|---------|-------------|
| **Actionability** | Turn scores into concrete next steps |
| **Accessibility** | Point users to existing resources |
| **Motivation** | Provide a clear path forward |

## 2. Recommendation Logic

### 2.1 Core Principle

**Recommendations are triggered by gaps.**

A "gap" is defined as a dimension score below 60.

| Score Range | Action |
|-------------|--------|
| 0–40 | Critical gap — urgent action required |
| 41–59 | Gap exists — action recommended |
| 60–79 | Good — continuous improvement |
| 80–100 | Excellent — maintain and share |

### 2.2 Recommendation Categories

Each dimension has:

1. **Primary recommendations** — High priority, score below 40
2. **Secondary recommendations** — Medium priority, score 41–59
3. **Enhancement recommendations** — Low priority, score 60–79
4. **Maintenance recommendations** — Score 80–100

## 5. Priority Scoring

### 5.1 Recommendation Priority

Each recommendation is assigned a priority score:

| Priority | Score | Description |
|----------|-------|-------------|
| **Critical** | 10 | Urgent action needed |
| **High** | 8 | Address immediately |
| **Medium** | 5 | Address within 3-6 months |
| **Low** | 2 | Continuous improvement |
| **Maintenance** | 1 | Maintain excellence |

### 5.2 Priority Calculation

Total priority score = Dimension gap severity × Recommendation priority

| Dimension Score | Gap Severity | Factor |
|-----------------|--------------|--------|
| 0–40 | Critical | 3 |
| 41–59 | Gap | 2 |
| 60–79 | Minor gap | 1 |
| 80–100 | No gap | 0 |

### 5.3 Priority Example

**Example: Procurement Score = 35 (Critical Gap)**

| Recommendation | Priority | Gap Factor | Total Priority |
|----------------|----------|------------|----------------|
| Establish procurement policy | 10 | 3 | 30 (Critical) |
| Train staff on procurement | 8 | 3 | 24 (High) |
| Document supplier evaluation | 8 | 3 | 24 (High) |
| Create supplier code of conduct | 8 | 3 | 24 (High) |

**Example: Procurement Score = 65 (Minor Gap)**

| Recommendation | Priority | Gap Factor | Total Priority |
|----------------|----------|------------|----------------|
| Optimize category management | 5 | 1 | 5 (Medium) |
| Benchmark against leaders | 2 | 1 | 2 (Low) |
| Achieve ISO certification | 1 | 1 | 1 (Maintenance) |

## 6. Recommendation Personalization

### 6.1 Personalization Factors

Recommendations should be personalized based on:

| Factor | How It Affects Recommendations |
|--------|--------------------------------|
| Sector | Industry-specific resources |
| Region | Location-specific programs |
| Company Size | Appropriate scale of resources |
| Maturity Level | Progression-appropriate advice |

### 6.2 Personalization Rules

**By Sector:**

| Sector | Additional Focus |
|--------|------------------|
| Mining | Safety, mining regulations, specialized suppliers |
| Agriculture | Seasonality, storage, export requirements |
| Manufacturing | Production planning, quality standards |
| Retail | Supply chain visibility, inventory management |
| Services | Service delivery, quality assurance |

**By Region:**

| Region | Additional Focus |
|--------|------------------|
| Gaborone | Access to training, networks, and programs |
| Francistown | Border logistics, transport corridors |
| Maun | Remote logistics, tourism supply chains |
| Other | Distance to resources, tailored solutions |

**By Company Size:**

| Size | Additional Focus |
|------|------------------|
| Micro (1–5 employees) | Simple solutions, low-cost resources |
| Small (6–50 employees) | Scalable solutions, accessible training |
| Medium (51–200 employees) | Structured solutions, partnerships |

## 7. Implementation Logic

### 7.1 Generate Action Plan

When a user completes the assessment, the system should:

1. Calculate dimension scores
2. Identify gaps (scores below 60)
3. Map gaps to recommendations
4. Prioritize recommendations
5. Personalize recommendations
6. Generate action plan

### 7.2 Action Plan Structure

The action plan should include:

| Section | Content |
|---------|---------|
| **Executive Summary** | Overall readiness score and maturity level |
| **Top 5 Recommendations** | Highest priority actions |
| **Dimension-by-Dimension** | Detailed recommendations for each dimension |
| **Resource Guide** | Where to find help |
| **Timeline** | Suggested timeframe for each action |
